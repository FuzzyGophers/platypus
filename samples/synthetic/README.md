# Synthetic fixtures (fictional — safe to publish)

Entirely **made-up** SDSx00 files used by the tests that need concrete,
location-specific assertions. No real place, person, or coordinate appears here —
the state is "Example State" (StateId 90), counties are Alpha/Bravo/Cedar
(9001/9002/9003), and coordinates are neutral high-plains points.

They reproduce the real on-card **structure** (record widths, column positions,
the favorites dialect, the `AreaCounty [owner-id, county-tag]` layout, a
`Rectangles`-shaped group, the favorites-only `DQKs_Status` / `BandPlan_P25`
records) so they exercise the same code paths as real card data — without
embedding anyone's location.

Generated by `generate.py` (re-run it to regenerate). Files:

| File | Purpose |
|---|---|
| `s_000090.hpd` | Multi-county/agency state HPDB. 4 systems; `by_county(9001)`→2, `(9002)`→2, `(9003)`→1, `(9101)`→0 (agency id is not a county — proves the field-2 rule) |
| `f_example.hpd` | Favorites dialect (blanked ids, no area tags) incl. `DQKs_Status`, `BandPlan_P25`, `Rectangle` |
| `hpdb.cfg` | Small county master (`CountyInfo`: Alpha=9001, Bravo=9002, Cedar=9003) |
| `f_list.cfg` | Favorites index (`F-List`) |

The **real** card fixtures live in `../BCDx36HP/` and are still exercised by the
byte-exact round-trip gate; this synthetic set is what the named, value-asserting
tests reference so the test code carries no real location.
